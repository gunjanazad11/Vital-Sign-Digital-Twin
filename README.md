## 📖 Overview

Continuous patient monitoring systems often suffer from noisy sensor readings, missing values, motion artifacts, and intermittent signal failures. These unreliable measurements can lead to inaccurate clinical decisions and false alarms.

This project presents a **Reliability-Aware Self-Correcting Digital Twin** that continuously validates physiological signals using a **GRU (Gated Recurrent Unit)** deep learning model. Instead of directly trusting incoming sensor data, the system reconstructs expected vital signs, estimates their reliability, automatically corrects faulty readings, and computes a **Health Risk Index (HRI)** to support clinical decision-making.

The project combines time-series forecasting, reliability estimation, self-correction, and health risk assessment into a single intelligent healthcare monitoring pipeline.

# 🧠 Methodology

### 1. Data Preprocessing

- Cleaning raw physiological signals
- Missing value interpolation
- Feature normalization
- Time-series preparation

### 2. Fault Injection

Artificial corruption is introduced into the signals using:

- Gaussian noise
- Missing values
- Random spikes

This enables the model to learn how to identify unreliable measurements.

### 3. GRU Model

A Gated Recurrent Unit (GRU) neural network learns temporal dependencies in patient vital signs and predicts the expected physiological values.

### 4. Reliability Estimation

Each incoming observation is compared against the GRU prediction.

A reliability score is calculated based on prediction error.

High reliability → original value retained.

Low reliability → corrected value generated.

### 5. Self-Correction

If the reliability score falls below a predefined threshold, the noisy sensor value is replaced with the predicted value generated by the GRU model.

### 6. Health Risk Index

Corrected vital signs are combined into a weighted Health Risk Index.

Patients are categorized into:

- 🟢 Normal
- 🟡 Warning
- 🔴 Critical

Clone the repository

```bash
git clone https://github.com/YOUR_USERNAME/HEALTH_INFO.git
```

Move into the project directory

```bash

```
